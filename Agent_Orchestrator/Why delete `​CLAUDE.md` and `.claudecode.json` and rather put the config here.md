Consolidating your behavioral configurations from `CLAUDE.md` and `.claudecode.json` directly into the orchestration harness’s system prompt is optimal for three core reasons:

* **Maximized Token Efficiency:** Claude Code natively reads workspace files like `CLAUDE.md` on every single turn and tool execution. By moving behavioral constraints out of physical files and directly into the harness injection loop, you eliminate redundant file-reading operations and slash input token overhead.
* **Unified Source of Truth:** Centralizing all rules into a single file (like `agent-config.json`) removes conflicting or duplicate instructions. This ensures that global constraints—such as strict token-saving brevity—and role-specific contexts (coding vs. assessment) are cleanly concatenated and fed to the model exactly once.
* **Stronger Instruction Adherence:** LLMs process guidelines passed via systemic injection (the system prompt) as absolute behavioral guardrails. Conversely, guidelines left inside markdown files are treated as external reference documentation, which requires more cognitive processing and increases the risk of the agent drifting from instructions during long, multi-turn tasks.
